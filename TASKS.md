## Phase 1 — Fork and run
**Goal:** A working vault with Gemini CLI driving real ingestion within one session.

- [ ] Fork `NicholasSpisak/second-brain` to your GitHub account
- [ ] Clone locally: `git clone git@github.com:[your-username]/second-brain.git`
- [ ] Install Gemini CLI: `npx @google/gemini-cli` — authenticate with Google account
- [ ] Install the skills: `npx skills add NicholasSpisak/second-brain`
- [ ] Run the setup wizard inside Gemini CLI: type `/second-brain`
  - Vault name: `poovi-second-brain`
  - Domain: Lloyd's insurance / AI engineering / podcast production
  - Agent: Gemini CLI
- [ ] Copy `GEMINI.md` (this repo) into the vault root — replace any `CLAUDE.md` generated by the wizard
- [ ] Initialise Git tracking on the vault: `git init && git add . && git commit -m "init: vault scaffold"`
- [ ] Install Obsidian and open the vault folder as a vault
- [ ] Drop 3–5 test sources into `raw/`:
  - One article (web clip from Obsidian Web Clipper)
  - One YouTube record from the liked-video database (manually export one row as JSON)
  - One existing document (a Mahabharata Moments episode note or a Lloyd's market article)
- [ ] Run `/second-brain-ingest` on each source and verify wiki pages are created
- [ ] Confirm `wiki/index.md` and `wiki/log.md` are populated
- [ ] Browse the Obsidian graph view — verify wikilinks are rendering

**Done when:** Three sources ingested, graph view shows linked pages, Git history shows one commit per ingest.

## Phase 2 — YouTube database integration
**Goal:** Pipe the existing YouTube liked-video database into the vault as a first-class source type.

- [ ] Export the YouTube liked-video database to a flat JSON file: `raw/youtube/liked-videos.json`
  - Schema per record: `{ "url", "title", "keywords", "transcript", "llm_summary" }`
- [ ] Write `scripts/youtube_to_raw.py`:
  - Reads `liked-videos.json`
  - For each record, writes a markdown file to `raw/youtube/[slug].md` using the template:
    ```
    ---
    title: "[title]"
    url: "[url]"
    keywords: [comma list]
    source_type: youtube
    ---
    ## Summary
    [llm_summary verbatim]

    ## Transcript
    [transcript — truncated to 8,000 words if longer]
    ```
  - Skips records already present in `raw/youtube/` (idempotent)
- [ ] Run the script: `python scripts/youtube_to_raw.py`
- [ ] Batch-ingest the YouTube records: `/second-brain-ingest raw/youtube/`
  - Watch for: speaker entities being created, concept pages for recurring themes
- [ ] Run `/second-brain-lint` — fix any broken wikilinks or missing index entries
- [ ] Verify the Obsidian graph now shows YouTube-sourced entity pages connecting to concept pages

**Done when:** All YouTube records are ingested, speaker and concept pages exist, graph is populated.

## Phase 3 — Gemini API pipeline
**Goal:** Move ingestion from interactive CLI to a Python script calling the Gemini API directly, giving you programmatic control and batch capability.

- [ ] Install the Python SDK: `pip install google-genai`
- [ ] Create `scripts/ingest.py`:
  - Accepts a file path or folder as argument
  - Calls `gemini-2.5-pro` for long sources (>10,000 words), `gemini-2.5-flash` for short
  - Uses schema-guided structured output (JSON) to extract: entities, concepts, key claims, contradictions
  - Writes wiki pages using the templates in `GEMINI.md`
  - Appends to `wiki/log.md` and updates `wiki/index.md`
  - Commits each write: `git commit -m "ingest: [title]"`
- [ ] Create `scripts/lint.py`:
  - Scans all wiki pages for broken wikilinks, orphan pages, missing frontmatter
  - Writes `output/lint-[date].md`
  - Optional: uses Gemini Search Grounding to flag stale claims on pages older than 180 days
- [ ] Test both scripts on 10 new sources
- [ ] Add `requirements.txt` with pinned dependencies
- [ ] Add a `.env.example` with `GEMINI_API_KEY=` and `GEMINI_MODEL=gemini-2.5-pro`

**Done when:** Ingestion runs headlessly via `python scripts/ingest.py raw/articles/` without touching Gemini CLI.

## Phase 4 — Staleness detection and synthesis
**Goal:** The wiki maintains itself — stale pages are flagged, cross-source synthesis is generated.

- [ ] Add staleness detection to `scripts/lint.py`:
  - For any `wiki/sources/` page older than 180 days, use Gemini with Search Grounding enabled
  - Ask: "Are the key claims on this page still accurate as of today?"
  - Write flagged claims to `wiki/flagged/[slug]-stale.md` for Poovi's review
- [ ] Build `/second-brain-synthesise` as a Python script `scripts/synthesise.py`:
  - Accepts a topic string: `python scripts/synthesise.py "Lloyd's delegated authority trends"`
  - Loads all relevant concept and source pages as context
  - Produces a `wiki/synthesis/[slug].md` page using the synthesis template
  - Adds backlinks from referenced pages
- [ ] Run synthesis on three cross-cutting topics:
  - AI agent workflow design patterns
  - Lloyd's market digitalisation
  - One Mahabharata Moments thematic thread
- [ ] Schedule weekly lint as a cron job or n8n workflow:
  - `0 9 * * 1 python /path/to/scripts/lint.py` (Monday 09:00)

**Done when:** At least three synthesis pages exist, lint runs automatically, stale flags appear in `wiki/flagged/`.

## Notes

- **Model routing shortcut:** set `GEMINI_MODEL=gemini-2.5-flash` in `.env` during development to save cost. Switch to `gemini-2.5-pro` for final ingest runs and synthesis.
- **Rate limits on free tier:** 60 req/min, 1,000 req/day. Batch YouTube ingestion in groups of 20 with a 2-second sleep between calls.
- **Git is your audit log:** never skip the commit step. The diff history is what you show in interviews as evidence of reviewable AI edits.
- **KISS checkpoint:** if any task feels like it needs a sub-task list longer than 5 items, it's too big — split into two tasks.
