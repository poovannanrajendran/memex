## Definition
A formal document that outlines the purpose, features, functionality, and behaviour of a product, serving as a guide for development teams.

## Why it matters (in Poovi's context)
Using AI to generate a PRD helps ensure a thorough and detailed plan for a project, which is critical for guiding AI coding assistants effectively.

## Key properties or components
- Defines product scope
- Details features and functionality
- Specifies user behaviour
- Guides development
- Format can be markdown
